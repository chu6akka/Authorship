# Authorship MVP (легкая версия)

Легкое MVP-приложение для судебного лингвистического сравнения двух текстов:
- полностью идентичные формулировки;
- n-граммы;
- базовый синонимизм;
- кандидаты на перефраз;
- кандидаты на транслитерацию;
- маркеры интернет-коммуникации.

---

## 1) Если «файл Python не запускается» — рабочая пошаговая инструкция

> Ниже самый надежный путь запуска с нуля.

### Шаг 1. Проверить Python

```bash
python --version
```

Нужен Python 3.10+.

### Шаг 2. Перейти в папку проекта

```bash
cd /workspace/Authorship
```

### Шаг 3. (Рекомендуется) создать виртуальное окружение

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### Шаг 4. Запустить приложение

#### Вариант A — нормальный графический интерфейс (рекомендуется)

```bash
python authorship_gui.py
```

Дальше в окне:
1. Нажмите **«Выбрать»** для исследуемого текста.
2. Нажмите **«Выбрать»** для образца.
3. Нажмите **«Запустить анализ»**.
4. Нажмите **«Сохранить отчет»**, чтобы выгрузить `.md`.

#### Вариант B — запуск через командную строку (CLI)

```bash
python authorship_mvp.py --query tests/data/query.txt --sample tests/data/sample.txt --report report.md
```

После запуска будет создан файл `report.md`.

## 2) Возможные проблемы и быстрые решения

### Ошибка: `python: command not found`
Установите Python и добавьте его в PATH.

### Ошибка кодировки при чтении файлов
Сохраните входные файлы в UTF-8.

### Графический интерфейс не открывается на Linux
Убедитесь, что установлен Tkinter:

```bash
python -m tkinter
```

Если окно не открылось, установите пакет Tk для вашей ОС.

## 3) Что выдает программа

Отчет в Markdown с разделами:
- входные материалы;
- базовая статистика;
- совпадения и частные признаки;
- маркеры интернет-коммуникации;
- черновик формулировки выводов (обязательно проверяется экспертом).

> Важно: результат — вспомогательная аналитика, а не автоматический процессуальный вывод.
